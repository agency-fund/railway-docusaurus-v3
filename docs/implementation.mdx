# Implementation

Once you have defined the [theory of change](theory-of-change.mdx) for your intervention, you have to determine how you will implement your intervention.

## Modules

In order to implement the intervention, you have to identify the module at each level, i.e., the piece of the puzzle, required to enable the intervention.

For instance, the farmer advice intervention would have the following modules:

|   | Level                                                   | Module                       |
| - | ------------------------------------------------------- | ---------------------------- |
| 1 | Context-aware advice will be made available for farmers | Recommendations Engine       |
| 2 | Farmers will access the advice                          | Whatsapp Bot                 |
| 3 | The advice will influence farmer behaviour              | Farm Inputs Data Collection  |
| 4 | Farm yields will improve                                | Farm Outputs Data Collection |

:::info Feature in an app vs Entire app

Your intervention could be a feature in an existing app, or it could be an entirely new app/interface. For instance, the intervention in the example above uses Whatsapp as the interface. Alternatively, the feature could have been a chatbot in an existing mobile app. Either way, levels 3 and 4 have to be considered and measured the same way because they are a consequence of the intervention.

:::

## User funnel

For the Whatsapp bot.



- Instrument the product to capture events automatically

- Use the events data to produce two metrics: activation (used once), and retention (used repeatedly)

- Look for patterns in the data, and talk to users to identify opportunities for improvement

- Test these ideas for improvement against these metrics with an A/B test
